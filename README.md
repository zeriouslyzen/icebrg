# ICEBURG: Enterprise AGI Platform

ICEBURG is a comprehensive Enterprise AGI Platform that provides unified access to advanced artificial general intelligence capabilities, autonomous software generation, and multi-agent civilization simulation.

## Overview

ICEBURG transforms complex AGI research into a unified, production-ready platform with enterprise-grade features including:

- **Unified Interface**: Single entry point with auto-mode detection for research, chat, software generation, and AGI civilization simulation
- **Global Middleware System**: Platform-wide hallucination detection and emergence tracking automatically applied to all agents, with global learning and pattern sharing
- **Enhanced Secretary Agent**: AGI-like capabilities with persistent memory, tool calling, multimodal processing, blackboard integration, efficiency optimizations, goal-driven autonomy (multi-step task planning), and self-updating knowledge base
- **One-Shot App Generation**: Complete macOS app generation pipeline with build, sign, notarize, and DMG creation
- **AGI Civilization Simulation**: Persistent multi-agent societies with social learning and emergent behaviors
- **Physiological Interface**: Real-time heart rate, breathing, stress monitoring with Earth connection and consciousness amplification
- **Business Mode**: Agent economy with individual wallets, USDC payments, and revenue tracking
- **Enhanced Swarm Architecture**: 6 swarm types with semantic routing and dual-audit mechanisms
- **Visual Generation**: Multi-platform UI generation (HTML5, React, SwiftUI) with red team validation
- **CIM Stack Architecture**: 7-layer consciousness integration with intelligent prompt interpretation
- **Virtual Scientific Ecosystems**: 3 research institutions with experimental design and hypothesis testing
- **Tesla-Style Learning**: End-to-end energy efficiency and hardware optimization
- **Instant Truth System**: Pattern recognition, breakthrough detection, and suppression detection
- **Enterprise Security**: SSO integration, DLP, access control, and audit logging
- **Multi-Platform Templates**: Unity, Unreal Engine, React Native, Android Studio support
- **Real-Time Monitoring**: Bottleneck detection and auto-healing capabilities

## Quick Start

### Installation

```bash
# Clone the repository
git clone https://github.com/your-org/iceburg.git
cd iceburg

# Install dependencies
pip install -r requirements/requirements_elite_financial.txt

# Initialize configuration
cp config/iceburg_unified.yaml.example config/iceburg_unified.yaml
```


## Modes

ICEBURG supports multiple modes to trade off latency and depth:

- Instant: sub-second reflexive outline (template + retrieval). No heavy agents.
- Fast: 1â€“3s beam-ranked retrieval + small SLM (configurable via `ICEBURG_FAST_MODEL`). Tight context and tokens. Skips heavy agents.
- Balanced: Standard protocol with parallelized deliberations and caching. Moderate depth in tens of seconds.
- Deep: Full multi-agent protocol (Surveyor/Dissident/Archaeologist â†’ enhanced deliberations â†’ Synthesist/Oracle). Highest depth.

Fast-path specifics: beam-scored reflexive routing, provider and Chroma singletons, hardware optimizer skipped, optional local embedding fallback for fast mode.

- `ICEBURG_PREFER_VLLM=1` â€“ prefer vLLM backend even if provider is set to ollama
- `ICEBURG_FORCE_FAST_LIMITS=1` â€“ clamp ctx/tokens globally in code paths (num_ctx<=1024, num_predict<=64)
- `ICEBURG_DISABLE_HARDWARE_OPT=1` â€“ skip Mac optimizer init
- `ICEBURG_DISABLE_REMOTE_EMBED_FAST=1` â€“ local deterministic embedding for fast mode
